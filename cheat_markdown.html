Markdown & Pandoc Cheatsheet

# For heading h1 ## For heading h1 etc..

Underline: Underline ========

Lists:

1. Item 2. Item

or * Item * Item Indent by 3 or more spaces for sub items.

Formatting:

*italics* or _italics_ **bold** or __bold__ One can also combine underscores and asterisk. ~~Strike through~~

Links:

[I'm an inline-style link](https://www.google.com)
[I'm an inline-style link with title](https://www.google.com "Google's Homepage")
[I'm a reference-style link][Arbitrary case-insensitive reference text]
[I'm a relative reference to a repository file](../blob/master/LICENSE)
[You can use numbers for reference-style link definitions][1]
Or leave it empty and use the [link text itself]
Some text to show that the reference links can follow later.
[arbitrary case-insensitive reference text]: https://www.mozilla.org
[1]: http://slashdot.org
[link text itself]: http://www.reddit.com

Code:

Inline code uses back-ticks before and after. For code blocks use three back ticks in a line followed by block followed by three back-ticks.

Tables:

Colons can be used to align columns.
| Tables        | Are           | Cool  |
| ------------- |:-------------:| -----:|
| col 3 is      | right-aligned | $1600 |
| col 2 is      | centered      |   $12 |
| zebra stripes | are neat      |    $1 |

The outer pipes (|) are optional, and you don't need to make the raw Markdown line up prettily. You can also use inline Markdown.

Markdown | Less | Pretty
--- | --- | ---
*Still* | `renders` | **nicely**
1 | 2 | 3

Blockquotes:

use the > symbol for blocking a line

Inline HTML:

Use raw html tags.

Horizontal Rule:

Three or more ... or --- or *** in a new line

Pandoc Command

pandoc -s -f markdown -t html cheat_markdown -o cheat_markdown.html

To generate html

pandoc -s -f markdown -t latex cheat_markdown -o cheat_markdown.tex

To generate standalone latex document.
